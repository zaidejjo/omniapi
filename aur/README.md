# AUR Packages for OmniAPI

This directory contains PKGBUILD templates for Arch User Repository (AUR) packages.

## Packages

| Package | Description | Source |
|---------|-------------|--------|
| `omniapi-desktop-bin` | Desktop GUI app (AppImage from GitHub releases) | `OmniAPI_${ver}_amd64.AppImage` |
| `omniapi-cli-bin` | CLI/TUI binary `oma` | `oma-linux-x64` |

## CI Automation

A GitHub Actions workflow (`.github/workflows/release.yml`) publishes both AUR packages
automatically whenever a `v*` tag is pushed:

1. **`cli` job** — builds `oma` binary for Linux, Windows, macOS
2. **`desktop` job** — builds Tauri desktop app (deb, AppImage, msi, dmg, etc.)
3. **`aur-publish` job** — after both complete, pushes updated PKGBUILD + `.SRCINFO` to AUR

### Required secrets

Set these in the repo → Settings → Secrets and variables → Actions:

| Secret | Purpose |
|--------|---------|
| `AUR_SSH_PRIVATE_KEY` | SSH key registered at https://aur.archlinux.org/account (`~/.ssh/authorized_keys`) |

The key must be added to your AUR account before CI can push.

### What the pipeline does for each AUR package

1. Clones `ssh://aur@aur.archlinux.org/<pkgname>.git`
2. Copies the PKGBUILD template from `aur/<pkgname>/PKGBUILD`
3. Replaces `YOUR_GITHUB_USERNAME/omniapi` with the actual repo URL
4. Updates `pkgver` to match the released tag
5. Downloads the corresponding Linux artifact from the release via `gh release download`
6. Computes `sha256sum` and writes it into the PKGBUILD
7. Generates `.SRCINFO` by sourcing the PKGBUILD bash variables (no `makepkg` needed)
8. Commits and pushes to AUR

> **Note**: The CI runs on `ubuntu-latest` which lacks `makepkg` / `base-devel`.
> The `.SRCINFO` is generated by `aur/generate-srcinfo.sh` which sources the PKGBUILD
> and exports its variables in the format AUR expects. This avoids needing `makepkg`
> on the runner.

## Manual publishing (fallback)

If CI fails or you need to publish outside a release:

```bash
VER="0.2.0"
REPO_URL="https://github.com/zaid/omniapi"

for pkg in omniapi-desktop-bin omniapi-cli-bin; do
  cd aur/$pkg

  # Update version + URL
  sed -i "s|YOUR_GITHUB_USERNAME/omniapi|zaid/omniapi|" PKGBUILD
  sed -i "s/^pkgver=.*/pkgver=${VER}/" PKGBUILD
  sed -i "s/^pkgrel=.*/pkgrel=1/" PKGBUILD

  # Download artifact and compute checksum
  curl -LO "$REPO_URL/releases/download/v$VER/$(source PKGBUILD && echo ${source[0]})"
  sed -i "s|^sha256sums=.*|sha256sums=('$(sha256sum OmniAPI* oma-linux* 2>/dev/null | cut -d' ' -f1)')|" PKGBUILD

  # Generate .SRCINFO and commit
  ../../aur/generate-srcinfo.sh PKGBUILD
  cd ../..
done
```

## Template placeholders

Before publishing, replace these placeholders in all PKGBUILD files:

- `YOUR_NAME` — your real name
- `YOUR_EMAIL` — your email
- `YOUR_GITHUB_USERNAME` — the GitHub account hosting the omniapi repo

The CI replaces `YOUR_GITHUB_USERNAME/omniapi` automatically from
`${{ github.repository }}`, but you must fill in `YOUR_NAME` and `YOUR_EMAIL`
manually in the PKGBUILD templates.
